# Privacy & Security Statement

This repository is a **public, documentation-only case study**. It is designed to demonstrate
product and engineering work **without** exposing anything proprietary or sensitive.

## What this repository contains
- Narrative documentation (README, case study, sanitized architecture, labeled metrics).
- A generalized architecture diagram, a public user-flow diagram, and a product timeline.
- Screenshots **only** after they have been reviewed and sanitized (see below).

## What it deliberately does **not** contain
- **No production source code.** The Midnight codebase is private.
- **No user data.** No names, emails, schools, messages, or any user-level records.
- **No secrets.** No API keys, tokens, credentials, environment variables, or project identifiers.
- **No security-sensitive detail.** No endpoint names, database schemas, auth/authorization
  rules, internal infrastructure, or proprietary prompts and routing logic.
- **No confidential business information** and no discussion of internal team/personnel matters.

## Screenshots policy
Any screenshots added later must:
- Use **sample or anonymized content** — no real user names, emails, school info, or private
  conversations.
- Exclude admin dashboards, internal analytics IDs, API keys, and proprietary prompts.
- Be reviewed against the pre-release checklist before being committed.

## Metrics policy
- All metrics are **aggregated**; no user-level data is published.
- Reported-but-unverified numbers are labeled as such and kept separate by product era
  (the beta vs Midnight). See [METRICS.md](METRICS.md).

## Architecture policy
The architecture is **intentionally generalized**. Specific implementation details have been
omitted because the production system is proprietary. See [ARCHITECTURE.md](ARCHITECTURE.md).
